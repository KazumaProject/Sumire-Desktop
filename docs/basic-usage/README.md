# Sumire IME 基本操作

この README は、`ComposingText`、`Preedit`、`SpaceKey` の動き、通常変換とライブ変換の違いをまとめたものです。

## 用語

### ComposingText

- コード上で、入力中の生データや表示用文字列を持つ内部バッファです。
- ユーザーが直接触るものではありませんが、`Preedit` やライブ変換の見え方の元になります。

### Preedit

- まだ確定していない文字列の画面表示です。
- かな入力中のひらがな表示も、ライブ変換中の先読み候補も、最終的にはこの未確定表示として見えます。

## 画面上の見え方

### かな入力中

- 変換前の `Preedit` は入力中文字列として表示されます。
- 通常は下線付きの未確定表示です。

例:

```text
きょうは
```

### 変換済みだが未確定

- 変換候補が出たあとの未確定文字列は、変換済み表示になります。
- フォーカスしていない文節は青い下線付きです。

### フォーカス中の文節

- 文節選択に入ると、注目中の文節だけ強調表示されます。
- 実装上は「白文字 + 青背景」で描画されます。

例:

```text
[今日]は
```

### ひらがな補助ビュー

- ライブ変換プレビュー ON かつ補助ビュー ON のとき、変換候補の横に元のひらがなが小窓で出ます。
- 変換待ち中は小さな青いインジケータが付きます。

## 通常変換の流れ

ライブ変換プレビューが OFF のときの基本動作です。

1. ローマ字またはかなを入力すると、まずひらがなの `Preedit` が出ます。
2. `Space` 1 回目で全体を変換します。
3. `Space` 2 回目で文節選択に入ります。
4. その後の `Space` で、現在フォーカス中の文節の候補を次へ送ります。
5. `Enter` で文節を確定し、未確定文節が残っていれば次の文節へ進みます。
6. 最後の文節まで確定すると、全体が確定します。

イメージ:

```text
入力中      : きょうは
Space 1回目 : 今日は
Space 2回目 : [今日]は
Space 3回目 : [こんにち]は   ← 現在文節の別候補
```

## ライブ変換の流れ

ライブ変換プレビューが ON のときの基本動作です。

1. 入力中から第1候補が `Preedit` に重ねて表示されます。
2. 必要なら補助ビューで元のひらがなを確認できます。
3. `Space` 1 回目でそのまま文節選択に入ります。
4. 以後の `Space`、`←`、`→`、`Enter` は通常変換の文節選択時と同じです。

イメージ:

```text
入力中      : 今日は   （未確定。内部では読みを保持）
Space 1回目 : [今日]は
```

通常変換との違いは、`Space` 1 回目の役割です。通常変換では「まず全体変換」、ライブ変換では「すぐ文節選択」に入ります。

## キー操作まとめ

- `Space`: 変換開始、または現在文節の次候補へ移動します。
- `Enter`: 現在の未確定内容を確定します。文節選択中は現在文節を確定して次へ進みます。
- `←` / `→`: 文節選択中はフォーカス文節を移動します。
- `↑` / `↓`: 候補ウィンドウが出ているとき、前候補 / 次候補へ移動します。
- `Esc`: 変換をキャンセルして `Preedit` に戻します。
- `Backspace` / `Delete`: 変換中はキャンセル、通常の `Preedit` 中は文字削除です。

## 文節を切り直したいとき

- `Shift + Space` で文節切り直し候補に入れます。
- この状態では `Space` や `←` / `→` で切り方候補を回し、`Enter` で適用します。
- `Esc` で文節切り直しを抜けて、通常の文節選択に戻れます。

## 覚えておくとよい最小手順

- 通常変換: 入力 -> `Space` -> `Space` -> `Enter`
- ライブ変換: 入力 -> `Space` -> `Enter`
- 候補違いを見たい: 文節選択中に `Space` または `↑` / `↓`
- やり直したい: `Esc`
